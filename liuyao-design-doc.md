## 一、产品概述

### 1.1 产品定位

**六爻玄机**是一款面向易学爱好者与传统文化用户的智能占卜 Web 应用，融合传统六爻纳甲预测法与现代 AI 大语言模型，提供专业排盘、智能解卦、对话追问三位一体的完整体验。

### 1.2 核心价值主张

| 维度 | 传统占卜软件 | 六爻玄机 |
|------|-------------|---------|
| 解卦方式 | 固定文本输出 | AI 实时对话，追问不限 |
| 起卦方法 | 单一方式 | 数字 / 铜钱 / 时间多选 |
| 用户留存 | 无激励机制 | 7天签到福利体系 |
| 开放能力 | 封闭 | 提供完整 Open API |

### 1.3 目标用户

- **核心用户**：传统文化爱好者、六爻学习者、易学研究者
- **扩展用户**：对命理感兴趣的泛大众用户
- **开发者用户**：希望接入六爻排盘 / AI 解卦能力的第三方开发者

## 五、六爻起卦模块

### 5.1 表单字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 问事标题 | 文本 | 是 | 用户所问之事 |
| 性别 | 单选 | 否 | 影响六亲用神判断 |
| 起卦时间 | 时间戳 | 是 | 默认当前时间，支持修改 |
| 地区 | 选择 | 是 | 用于计算真太阳时 |
| 真太阳时 | 开关 | 否 | 开启后自动换算 |
| 起卦方式 | 选择 | 是 | 数字/铜钱/时间 |
| 数字 | 数字 | 条件 | 数字起卦时必填 |
| 动爻加时辰 | 开关 | 否 | 单数字时影响动爻取法 |

### 5.2 起卦算法（数字起卦）

**多位数规则：**
- 奇数位数（如7位）：前3位为上卦，后4位为下卦
- 偶数位数：前半为上卦，后半为下卦
- 动爻：所有数字之和 ÷ 6 取余

**单位数规则：**
- 上卦：此数 ÷ 8 取余
- 下卦：当前时辰数 ÷ 8 取余
- 动爻：(此数 + 时辰数) ÷ 6 取余（开启加时辰时）

**八卦对应：** 乾1 兑2 离3 震4 巽5 坎6 艮7 坤8

### 5.3 卦象展示结构

```
┌─────────────────────────────┐
│  本卦名称 → 变卦名称          │
│  （卦型：六合/六冲/游魂/归魂）  │
├─────────────────────────────┤
│  六爻图形（上卦3爻 + 下卦3爻） │
│  每爻：六兽 飞神 爻名 世/应标记 │
├─────────────────────────────┤
│  详细爻表（6行）              │
│  六兽 | 伏神 | 爻名五行 | 旺衰 │
├─────────────────────────────┤
│  卦身 | 世身 | 内外卦特征      │
└─────────────────────────────┘
```

### 7.3 POST /ai/interpret — AI 解卦对话

**请求参数**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| hexagram_id | string | 是 | 排盘返回的 ID |
| mode | string | 是 | 见下方枚举 |
| message | string | 条件 | mode=general 时必填 |
| history | array | 否 | 对话历史 `[{role, content}]` |
| stream | boolean | 否 | 是否流式输出，默认 `false` |

**mode 枚举值**

| 值 | 含义 |
|----|------|
| `portrait` | 人物画像分析 |
| `timing` | 应期预测 |
| `yongshen` | 用神分析 |
| `liuqin` | 六亲关系 |
| `feifu` | 飞伏关系 |
| `general` | 自定义追问 |

**流式响应（SSE，stream=true）**

```
data: {"delta":"此卦"}
data: {"delta":"离为火"}
data: {"delta":"，主事明朗，应期..."}
data: [DONE]
```

**普通响应**

```json
{
  "code": 200,
  "data": {
    "reply": "此卦离火旅卦，用神财爻己酉金居四爻...",
    "quota_remaining": 4
  }
}
```

### 7.6 错误码

| Code | 含义 |
|------|------|
| 200 | 成功 |
| 400 | 参数错误 |

## 九、数据流与状态管理

### 9.1 前端状态

```javascript
AppState = {
  user: { id, gender, location },
  signin: { streak, today_done, rewards },
  divination: {
    question, method, numbers, timestamp,
    result: { hexagram_id, lines, gua_info }
  },
  chat: {
    hexagram_id,
    history: [{ role, content }],
    quota_remaining
  }
}
```

### 9.2 数据流

```
用户填写表单
    ↓
POST /divination/liuyao
    ↓
hexagram_id + 卦象数据 → 渲染结果
    ↓
用户点击快捷键 / 输入追问
    ↓
POST /ai/interpret (带 history)
    ↓
SSE 流式 → 实时渲染气泡
    ↓
history 追加 → 下轮请求
```
