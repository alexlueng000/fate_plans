## 新增表结构

### 1. emotion_records（情绪记录表）

核心表，存储用户每日情绪记录。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键，自增 |
| user_id | INT | 用户ID（外键 users.id） |
| record_date | DATETIME | 记录日期时间 |
| solar_term | VARCHAR(32) | 当前节气（如：立春、雨水） |
| wuxing_element | VARCHAR(16) | 五行属性（金/木/水/火/土） |
| emotion_score | SMALLINT | 情绪评分 1-10 |
| emotion_tags | VARCHAR(255) | 情绪标签（JSON数组字符串） |
| content | TEXT | 情绪记录正文 |
| ai_response | MEDIUMTEXT | AI回应内容（异步写入） |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

**索引**：
- PRIMARY KEY (id)
- INDEX idx_user_date (user_id, record_date)
- INDEX idx_created (created_at)

**外键**：
- user_id → users(id) ON DELETE CASCADE

---

### 2. exception_moments（例外时刻表）

记录用户的积极例外事件。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键，自增 |
| user_id | INT | 用户ID（外键 users.id） |
| emotion_record_id | INT | 关联的情绪记录ID（可空） |
| title | VARCHAR(255) | 例外时刻标题 |
| content | TEXT | 例外时刻详细描述 |
| moment_date | DATETIME | 例外时刻发生日期 |
| created_at | DATETIME | 创建时间 |

**索引**：
- PRIMARY KEY (id)
- INDEX idx_user_created (user_id, created_at)

**外键**：
- user_id → users(id) ON DELETE CASCADE
- emotion_record_id → emotion_records(id) ON DELETE SET NULL

---

### 3. value_actions（价值行动表）

记录用户的价值观和行动计划。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键，自增 |
| user_id | INT | 用户ID（外键 users.id） |
| emotion_record_id | INT | 关联的情绪记录ID（可空） |
| value_name | VARCHAR(128) | 价值观名称 |
| action_plan | TEXT | 行动计划 |
| status | SMALLINT | 状态：0=计划中, 1=进行中, 2=已完成 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

**索引**：
- PRIMARY KEY (id)
- INDEX idx_user_created (user_id, created_at)

**外键**：
- user_id → users(id) ON DELETE CASCADE
- emotion_record_id → emotion_records(id) ON DELETE SET NULL

## 执行步骤

### 前置检查

1. **确认数据库连接**
```bash
mysql -u root -p
USE fate;
SHOW TABLES;
```

2. **备份数据库（必须！）**
```bash
# 备份整个数据库
mysqldump -u root -p fate > backup_fate_$(date +%Y%m%d_%H%M%S).sql

# 或者只备份 users 表（新表依赖此表）
mysqldump -u root -p fate users > backup_users_before_emotion.sql
```

### 执行迁移

**方法一：使用 MySQL 命令行**

```bash
cd fate
mysql -u root -p fate < migrations/003_add_emotion_tables.sql
```

**方法二：在 MySQL 客户端中执行**

```sql
USE fate;
SOURCE migrations/003_add_emotion_tables.sql;
```

### 验证迁移

```sql
-- 1. 检查表是否创建成功
SHOW TABLES LIKE '%emotion%';
SHOW TABLES LIKE '%exception%';
SHOW TABLES LIKE '%value%';

-- 2. 检查表结构
DESCRIBE emotion_records;
DESCRIBE exception_moments;
DESCRIBE value_actions;

-- 3. 检查外键约束
SELECT 
    CONSTRAINT_NAME, 
    TABLE_NAME, 
    COLUMN_NAME, 
    REFERENCED_TABLE_NAME, 
    REFERENCED_COLUMN_NAME
FROM INFORMATION_SCHEMA.KEY_COLUMN_USAGE
WHERE TABLE_SCHEMA = 'fate' 
  AND TABLE_NAME IN ('emotion_records', 'exception_moments', 'value_actions')
  AND REFERENCED_TABLE_NAME IS NOT NULL;

-- 4. 检查索引
SHOW INDEX FROM emotion_records;
SHOW INDEX FROM exception_moments;
SHOW INDEX FROM value_actions;

-- 5. 测试插入（可选）
INSERT INTO emotion_records (user_id, record_date, emotion_score, content)
VALUES (1, NOW(), 7, '测试记录');

SELECT * FROM emotion_records WHERE user_id = 1;

-- 清理测试数据
DELETE FROM emotion_records WHERE content = '测试记录';
```

### 回滚（如果需要）

⚠️ **警告**：回滚将删除所有心镜灯相关数据！

```bash
mysql -u root -p fate < migrations/003_rollback_emotion_tables.sql
```

## 常见问题

### Q1: 外键约束失败

**错误信息**：
```
ERROR 1215 (HY000): Cannot add foreign key constraint
```

**解决方案**：
1. 确认 `users` 表存在且有 `id` 字段
2. 确认 `users.id` 是 INT 类型
3. 检查字符集和排序规则是否一致

```sql
SHOW CREATE TABLE users;
```

### Q2: 表已存在

**错误信息**：
```
ERROR 1050 (42S01): Table 'emotion_records' already exists
```

**解决方案**：
迁移脚本使用了 `CREATE TABLE IF NOT EXISTS`，此错误不应出现。如果出现，说明表已创建，可以跳过或先删除旧表：

```sql
DROP TABLE IF EXISTS value_actions;
DROP TABLE IF EXISTS exception_moments;
DROP TABLE IF EXISTS emotion_records;
```

### Q3: 字符集问题

**错误信息**：
```
Incorrect string value: '\xE6\x84\x9F\xE5\x8F\x97' for column 'content'
```

**解决方案**：
确保数据库和表使用 utf8mb4 字符集：

```sql
ALTER DATABASE fate CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

## 性能优化建议

### 1. 索引优化

如果用户量大（>10万），考虑添加复合索引：

```sql
-- 优化按日期范围查询
ALTER TABLE emotion_records 
ADD INDEX idx_user_date_score (user_id, record_date, emotion_score);

-- 优化按状态查询价值行动
ALTER TABLE value_actions 
ADD INDEX idx_user_status (user_id, status);
```

### 2. 分区表（可选）

如果数据量预计超过100万条，考虑按月分区：

```sql
ALTER TABLE emotion_records 
PARTITION BY RANGE (YEAR(record_date) * 100 + MONTH(record_date)) (
    PARTITION p202604 VALUES LESS THAN (202605),
    PARTITION p202605 VALUES LESS THAN (202606),
    -- ... 按需添加
    PARTITION pmax VALUES LESS THAN MAXVALUE
);
```

### 3. 定期清理

建议定期归档或删除超过1年的数据：

```sql
-- 归档到历史表
INSERT INTO emotion_records_archive 
SELECT * FROM emotion_records 
WHERE record_date < DATE_SUB(NOW(), INTERVAL 1 YEAR);

-- 删除旧数据
DELETE FROM emotion_records 
WHERE record_date < DATE_SUB(NOW(), INTERVAL 1 YEAR);
```
